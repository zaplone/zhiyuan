# 产品表字段与前端对应表（含维护说明）

> 供**后台/独立站 API 维护人员**和**前端开发**共用：每个字段在后台叫什么、在哪展示、怎么填。  
> 表内「产品卡片」= 首页 6 款精选 + 底部流动画廊 + 产品列表页的卡片；「详情页」= 单产品详情页；「快速预览」= 点击卡片弹出的弹窗。  
> **说明**：产品列表页左侧筛选已取消，以下字段仅用于展示与标签，不再参与筛选。

---

## 一、必填 vs 可不填（降低填写难度）

维护人员**最少只需填 3 项**即可发布一条产品，其余均可不填（前端有默认或占位）。

| 是否必填 | 字段 | 说明 |
|----------|------|------|
| **必填** | name、model_code、safety_standard | 名称、型号、安全等级。缺一不可，否则列表/详情无法正常展示。 |
| **可不填（有默认值）** | moq、is_hot、is_new | 后台已设默认：MOQ 默认 `500 Pairs`，热销/新品默认不勾选。不填即用默认。 |
| **可不填（前端有占位）** | description、images、materials、industries、features、additional_certs、style、price_range、specs_extra | 空则详情页用占位文案或默认图；卡片/列表会尽量展示有值的项，没有就省略或默认。 |

**建议最少录入**：必填 3 项 + **至少 1 张图片**（否则卡片/详情缺图）。其余按需补全，不必一次填满。

## 三、扩展字段 specs_extra（JSON）

**specs_extra** 为产品表唯一扩展字段，用于规格说明及后续新增配置，**不填亦可**（前端用默认文案）。表结构不再为此类内容新增独立字段。

**建议 JSON 结构示例**（键名由前端约定，后端只按此结构存）：

```json
{
  "weight": "As per standard",
  "test_standard": "EN ISO 20345",
  "colors": "Black, Navy, Brown",
  "sizes": "36–48 (EU)"
}
```

| 键名 | 用途 | 详情页 | 快速预览 | 不填时 |
|------|------|--------|----------|--------|
| weight | 重量 | 规格表「重量」行 | — | 显示站点默认文案 |
| test_standard | 测试标准 | 规格表「测试标准」行 | — | 显示默认 |
| colors | 颜色 | 规格表「颜色」行 | — | 显示默认 |
| sizes | 尺码 | 规格表「尺码」行 | 尺码展示 | 显示 `36–48 (EU)` 或默认 |

**说明**：  
- **不需要单独存产品详情页链接**。详情页 URL 由前端根据 **model_code** 自动生成（如 `LY-8801` → `/en/products/ly-8801`），无需在 product 或 specs_extra 里再存 slug/链接。  
- specs_extra 随产品一起在后台保存即可，无需单独接口或单独保存。  
- 后续若需新增展示项（如包装说明、保质期等），在此 JSON 中加键即可，前端按 key 匹配展示，**无需再改表结构**。

## 五、关联表简要说明（维护时在后台选即可）

- **additional_certs**：来自 **Certification** 集合，在后台先维护好认证（如 SRC、WR、ESD），再在产品里多选。
- **industries**：来自 **Industry** 集合，先维护行业（如 Construction、Logistics），再在产品里多选。
- **features**：来自 **Feature** 集合，先维护功能（如 Slip Resistant、Waterproof），再在产品里多选。

这些字段在**产品卡片**里用于角标/标签；在**详情页**用于 Hero 标签、行业应用区块、卖点标签；在**快速预览**里用于角标和材质/功能列表。（左侧筛选已取消，不再参与筛选。）

## 七、维护时建议顺序（人类操作）

1. **最少发布**：只填 **name**、**model_code**、**safety_standard** 即可保存；建议再上传 **至少 1 张图片**。
2. **按需补全**：description、moq、materials、industries、additional_certs、features、style、price_range、is_hot、is_new 均可后补，不填也有默认或占位。
3. **扩展规格**：需要单独展示重量/测试标准/颜色/尺码时，在 **specs_extra** 里填 JSON（见第三节）；不填则前端用默认文案。
4. **多语言**：name、description、materials、specs_extra 已开 i18n，各语言各填一份即可。

这样表结构稳定，维护人员只需按「必填 3 项 + 建议 1 图」即可上线，其余按需填写。

## 附：产品详情页与静态构建

本站为 **SSG 静态站点**（`output: 'export'`）。产品详情页**不是**用户访问时动态拼接的，而是**构建时预生成**的：

1. **构建时**：`generateStaticParams()` 会请求独立站 API 拿到所有产品，用 `model_code` 生成 slug（如 `ly-8801`），得到所有 `(locale, slug)` 组合。
2. **对每个组合**：Next.js 预渲染一页，得到静态 HTML（如 `en/products/ly-8801/index.html`），并部署到 CDN。
3. **用户访问**：请求的即是这份已生成好的静态页，不再实时请求 API。  
因此**不需要在库里单独存详情页链接**，链接由 `model_code` 在构建时确定；新增/修改产品后需**重新构建并部署**，新详情页才会生效。
