## 1. 项目概览

### 1.1 项目定位

登泰劳保鞋独立站是一个**现代化的多语言 B2B 营销网站**：前端为静态站点（SSG），**业务数据在构建时从独立站后端 API 拉取**，实现展示与数据服务分离。

### 1.2 核心特点

- 🌍 **多语言支持**: 中文/英文，易于扩展其他语言
- ⚡ **极速访问**: 采用 SSG + 全球 CDN（Cloudflare Pages）
- 📝 **内容管理**: 产品、新闻、询盘等由**鹿原侧业务系统 / 后台**维护（不在本 Git 仓库）
- 🔄 **自动化部署**: 推送代码或 Webhook 触发 Cloudflare 重新构建后上线
- 💰 **低成本**: 静态托管免费为主，API 与服务器成本按团队实际部署计

### 1.3 主要功能模块

| 模块 | 功能 | 管理方式 |
|------|------|----------|
| 首页 | 品牌展示、产品推荐、新闻动态 | 业务后台 + 构建发布 |
| 产品展示 | 产品列表、分类、详情、筛选 | 业务后台（经独立站 API 提供数据） |
| 关于我们 | 公司介绍、发展历程、认证资质 | 代码内容 |
| OEM 服务 | 定制流程、案例展示 | 代码内容 |
| FAQ | 常见问题（部分接口数据） | 代码 + `NEXT_PUBLIC_API_URL` 站点 |
| 询盘表单 | 客户咨询提交 | 独立站 API `.../api/inquiries` |

### 1.4 本仓库代码范围

本 Git 仓库**仅包含**：

- `safetyshoe-frontend/`：Next.js 前端（静态导出）
- `1-重要文档/` 等说明文档

**独立站数据 API（生产）基地址**：

```text
https://luyuan.zwstone.cn/api
```

前端通过 `src/lib/siteApi.ts` 在**构建时与运行时**请求该服务（产品公开列表、工厂新闻、询盘等）。  
环境变量优先使用 `NEXT_PUBLIC_SITE_API_URL`；若历史环境仍配置 `NEXT_PUBLIC_STRAPI_URL`，代码会**兼容读取**（旧名与历史文档一致，**并非** Strapi CMS）。

后端服务（Nest 等）、数据库、部署方式**不在本仓库**维护。

## 2. 整体架构

### 2.1 架构图（逻辑）

```
                    ┌──────────────┐
                    │  用户浏览器   │
                    └──────┬───────┘
                           │ HTTPS
                           ▼
                    ┌──────────────┐
                    │ Cloudflare   │
                    │ Pages (CDN)  │
                    │ 静态 HTML/JS │
                    └──────┬───────┘
                           │
           构建时拉取数据   │
                           ▼
┌─────────────────────────────────────────────┐
│  独立站 API（生产）                          │
│  https://luyuan.zwstone.cn/api               │
│  · standalone/products/public                │
│  · factory-updates                           │
│  · api/inquiries 等                          │
└─────────────────────────────────────────────┘
                           ▲
                           │ 管理端 / ERP（团队内部）
                    ┌──────┴───────┐
                    │  GitHub      │
                    │  前端源码     │
                    └──────────────┘
```

### 2.2 三层说明

**展示层**：Cloudflare Pages 托管的静态资源，全球边缘节点访问。  
**数据层**：`https://luyuan.zwstone.cn/api` 提供 JSON；构建阶段 `next build` 会请求该 API 生成各语言静态页。  
**源码层**：GitHub 仅存前端仓库；数据与后台运维在鹿原侧。

## 4. 部署架构

### 4.1 前端（Cloudflare Pages）

**流程简述**：

1. 推送代码到 GitHub  
2. Cloudflare 拉取并执行构建  
3. 构建过程请求 `NEXT_PUBLIC_SITE_API_URL`（或兼容的 `NEXT_PUBLIC_STRAPI_URL`）拉取列表与详情所需数据  
4. 生成 `out/` 静态文件并发布到 CDN  

**关键配置**：

- **根目录**：`safetyshoe-frontend/`
- **构建命令**：`npm ci && npm run build`（以项目实际为准）
- **输出目录**：`out/`

**环境变量（示例）**：

```bash
NEXT_PUBLIC_SITE_API_URL=https://luyuan.zwstone.cn/api
NEXT_PUBLIC_API_URL=https://你的对外站点域名
NEXT_PUBLIC_SITE_URL=https://你的对外站点域名
```

说明：`NEXT_PUBLIC_API_URL` 仍用于站内部分路径（如 FAQ、联系接口），与「独立站数据 API」基地址可以不同，按现网配置为准。

### 4.2 独立站 API

- **生产基地址**：`https://luyuan.zwstone.cn/api`  
- 扩容、监控、证书、限流等由**服务端与运维**负责，不在本文档展开。

### 4.3 图片与静态资源

产品图等一般为 **HTTPS 绝对 URL**（由 API 返回）。若使用对象存储，需在存储侧配置 CORS 与域名策略（按现网环境执行）。

## 5. 日常运维

### 5.1 检查清单（建议）

**每日**：站点可访问；Cloudflare 最近一次构建是否成功。  
**每周**：构建失败告警；API 可用性（可对 `/api` 做健康检查，由服务端提供则按其文档）。  
**每月**：依赖与安全更新；域名与证书有效期。

### 5.2 监控指标（参考）

| 指标 | 说明 |
|------|------|
| 页面首屏 | 浏览器 / RUM |
| 构建耗时 | Cloudflare Pages 构建日志 |
| API 可用性 | 独立站 API 响应与错误率 |

### 5.3 备份

- **代码**：GitHub 版本历史。  
- **业务数据**：由鹿原侧数据库与备份策略负责。

## 6. 内容更新流程

### 6.1 产品与新闻

1. 在**业务后台**维护产品、新闻等内容（具体系统以团队为准）。  
2. 数据经独立站 API 对外提供。  
3. **重新生成静态站**：推送前端仓库触发构建，或通过 Cloudflare / GitHub Webhook 触发（与现网一致）。  
4. 构建完成后，用户看到新内容。

### 6.2 询盘

询盘提交走独立站 API（如 `POST .../api/inquiries`），后台查看方式以服务端与业务配置为准。

### 6.3 自动化更新（示意）

```
运营更新数据 → API 数据变化 → （Webhook）→ Cloudflare 构建 → 静态站更新
```

从保存到上线时间取决于构建队列，常见约数分钟。

## 7. 开发与上线流程

### 7.1 本地开发

```bash
cd safetyshoe-frontend
npm install
npm run dev
# 浏览器访问 http://localhost:3000
```

本地可在 `safetyshoe-frontend/.env.local` 中设置：

```bash
NEXT_PUBLIC_SITE_API_URL=https://luyuan.zwstone.cn/api
```

如需指向测试环境，改为测试基地址即可。

### 7.2 上线

```bash
npm run build
npx serve out   # 本地预览静态产物
git add . && git commit -m "feat: ..." && git push
```

### 7.3 提交信息规范（建议）

`feat:` / `fix:` / `docs:` / `style:` / `refactor:` / `chore:` 等。

## 8. 故障排查

### 8.1 网站打不开

1. Cloudflare Pages 项目是否最新成功部署。  
2. 自定义域名 DNS / SSL 是否正常。  
3. 强制刷新缓存（Ctrl+Shift+R）。

### 8.2 构建失败

1. 查看 Cloudflare 构建日志。  
2. 常见原因：TypeScript 错误、依赖安装失败、**构建时无法访问** `NEXT_PUBLIC_SITE_API_URL`（网络、防火墙、证书）。  
3. 本地执行 `npm run build` 复现。

### 8.3 数据不更新 / 空白

1. 确认 API 基地址环境变量与线上一致。  
2. 浏览器或构建日志中请求 `https://luyuan.zwstone.cn/api/...` 是否 200。  
3. 确认已重新触发**静态站构建**（仅改 API 数据不会自动改已发布的 HTML，除非配置了 Webhook 自动构建）。

### 8.4 图片不显示

检查返回的 URL 是否 `https://`、是否被浏览器拦截（混合内容、CORS）。
