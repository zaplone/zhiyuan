# 独立站首页 — 改版布局说明

> **前提（不变）**：Next.js 静态导出、`siteApi` 数据接口、Cloudflare 部署、后台与 API 契约均不改变。  
> **本文**：说明「若由零规划首页信息架构」时的区块顺序与意图；**当前仓库已按此顺序落地一版**（见下「与代码对应」）。

---

## 一、改版后的区块顺序（自上而下）

| 顺序 | 区块 | 作用（B2B 视角） |
|------|------|------------------|
| 1 | **Hero** | 价值主张 + 主 CTA（进目录）+ 次 CTA（联系/询盘锚点） |
| 2 | **TrustBar（信任条）** | 首屏下方立刻给「可验证实力」的摘要：认证、标准、产能、市场，降低跳出 |
| 3 | **精选产品 + 画廊** | 证明「我们卖什么」；同一 API 数据，强调浏览与转化 |
| 4 | **Why Choose Us** | 深度信任：制造 / 品控 / 物流三块故事 + 数据头图 |
| 5 | **OEM 导流条** | 把「要贴牌/定制的访客」导到 OEM 服务页，不与产品区抢注意力 |
| 6 | **工厂动态 / 新闻** | 时效与真实感，利于回访与 SEO |
| 7 | **FAQ + 联系条** | 扫清采购疑虑；底部 **#contact** 锚到邮件与 WhatsApp |

**原顺序**：Hero → WhyChooseUs → 产品 → 新闻 → FAQ。  
**新顺序**：把「产品」上提到信任条之后，让访客**先看到卖什么、再听故事**；OEM 单独成条，路径更清晰。

---

## 二、为何这样排（简要）

1. **首屏后紧跟 TrustBar**：避免「只有大图口号」的空窗感，符合采购商「先确认是不是正经工厂」的心理顺序。  
2. **产品前置**：目录与 API 不变，仅调整叙事顺序，提高「找型号」效率。  
3. **Why Choose Us 仍保留完整**：深度背书放在产品之后，给愿意滚动的人看细节。  
4. **OEM 独立横条**：B2B 里定制需求强，用一条轻量 CTA 指向 `/services/oem`，不增加新接口。  
5. **#contact**：底部深色条增加 `id="contact"`，与 Hero 次按钮、`/#contact` 一致，避免锚点失效。

## 三、与代码对应（便于维护）

| 文档区块 | 组件 / 位置 |
|----------|-------------|
| Hero | `src/components/Hero.tsx` |
| TrustBar | `src/components/TrustBar.tsx`，文案 `TrustBar`（en/zh/ru） |
| 精选产品 | `src/components/ProductCategories.tsx`（`hideFilters` 等 props 不变） |
| Why Choose Us | `src/components/WhyChooseUs.tsx` |
| OEM 导流 | `src/components/HomeOemCta.tsx`，文案 `HomeOemCta` |
| 新闻 | `src/components/FactoryNews.tsx` |
| FAQ + 联系 | `src/components/FAQAndContact.tsx`（`id="contact"` 在底部 CTA 容器） |

首页组装：`src/app/[locale]/page.tsx` 中按上表顺序引用。

---

## 四、后续可迭代（仍不改 API）

- TrustBar 四项可换成真实客户 logo 墙或「证书下载」链接（静态资源）。  
- Hero 高度在移动端可再收一点，避免首屏过长（仅样式调整）。  
- Why Choose Us 内三个视频若仍为同一占位 URL，建议逐步换成不同实拍片段，增强可信度。
